Тест требует создания локальной базы. Скрипт прилагается.
Запустить можно UWP (+WebServer);

Запуск андроидного приложения требует наличия WebServer в доступе телефона (т.е. либо это внешний сервер, либо локальный во внутренней сети по WiFi, в тесте не проверялось)


Не использовано в задаче:
Логирование, 
обработка ошибок, 
иверсия зависимости, 
ОРМ,
T-Sql (нормирование, оптимизация бд),
Unit tests,
Неучтены real time эффекты, если потребуется одновременно отправлять и получать данные с сервера
Логика сохранения в бд, тоже не идеальна, но для данной задачи, она работает верно


Продемонстрировано: 
Базовая структура проекта (она же основная скорее всего),
Сборка Shared демонстрирует принцип расширения общего функционала между Xamarin и WebServer проектами (.net standart dll),
Single responsibility в отношении классов,
базовый sql, 
понимание task (async await), 
разделение view и модели в клиентской части,
патерн NotifyPropertyChanged (без использования PostSharp),
классическая CamelCase C# стилистика


Проекты брались в оригинальном виде генерируемые Visual Studio 2017, поэтому структура проектов сохранена и дополнена своим кодом.
Проекты для дополнительных платформ iOS (и др.) не включены, т.к. у меня нет iPhone для тестирования и полноценной разработки, 
	а также аккаунта, для публикаций.
